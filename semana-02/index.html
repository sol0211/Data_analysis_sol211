Semana 02 · Caso Empresarial

¿Por qué cayeron
las ventas de NovaMart?

Un retailer online con operaciones en 5 ciudades detecta una caída del 23% en ventas. El problema: nadie puede explicarla porque los datos están sucios.

🏪 Contexto del caso
NovaMart E-commerce · Lima, 2024
NovaMart es una tienda online de electrónica y accesorios con operaciones en Lima, Arequipa, Cusco, Trujillo y Piura. Su equipo de datos recibe tres archivos exportados de tres sistemas distintos: el ERP de inventario, la plataforma de ventas online y el CRM de clientes. El gerente comercial exige un reporte urgente: "¿Por qué las ventas de marzo cayeron un 23% respecto a febrero?" — pero cuando el analista junior abre los datos, encuentra un caos total.
PROBLEMA 01
Fechas en 3 formatos
Cada sistema exporta fechas distintas
15/03/2024 · 2024-03-15 · Mar 15 24
PROBLEMA 02
Ventas duplicadas
Pedidos cancelados y reprocesados aparecen dos veces
ORDER-4521 aparece 2 veces
PROBLEMA 03
Clientes sin región
40% de registros sin ciudad o con abreviaturas
NULL · LIM · lima · LIMA
PROBLEMA 04
Montos en soles y dólares
Sin indicador de moneda en cada fila
150.00 ≠ 150.00 (S/ vs $)

🎯 Por qué este modelo
El esquema estrella es el estándar en warehouses como BigQuery. Permite hacer JOINs eficientes, agregar métricas por dimensión, y escalar sin reescribir queries. La tabla de hechos tiene los números; las dimensiones tienen los contextos.

Semana 02 · SQL · BigQuery

Integrar datos
con SQL en BigQuery

Una vez los datos están limpios en BigQuery, usamos SQL para integrar las tablas y construir la vista analítica que responderá las preguntas del negocio.

SQL · BigQuery
-- Vista analítica: ventas limpias con contexto de cliente y producto CREATE OR REPLACE VIEW novamart.analytics.v_ventas_limpias AS SELECT v.order_id, v.fecha_venta, FORMAT_DATE('%Y-%m', v.fecha_venta) AS mes_año, c.ciudad, c.segmento, p.categoria, p.nombre AS producto, v.monto_soles, v.estado FROM novamart.raw.fact_ventas v LEFT JOIN novamart.raw.dim_clientes c ON v.cliente_id = c.cliente_id LEFT JOIN novamart.raw.dim_productos p ON v.producto_id = p.producto_id WHERE v.estado = 'completado' AND v.monto_soles IS NOT NULL AND v.fecha_venta BETWEEN '2024-01-01' AND '2024-03-31'; -- Query de análisis: ventas por mes y ciudad SELECT mes_año, ciudad, COUNT(order_id) AS total_pedidos, SUM(monto_soles) AS ventas_totales, AVG(monto_soles) AS ticket_promedio FROM novamart.analytics.v_ventas_limpias GROUP BY 1, 2 HAVING ventas_totales > 0 ORDER BY mes_año, ventas_totales DESC;
💡 Conceptos clave en este query
LEFT JOIN une tablas manteniendo todos los registros de ventas aunque falte info del cliente.
GROUP BY agrupa para calcular totales por mes y ciudad.
HAVING filtra grupos después de agregar (a diferencia de WHERE que filtra antes).
FORMAT_DATE extrae el período mes-año para comparar temporadas.

Semana 02 · Métricas de Negocio

Construir los
KPIs básicos

Con los datos limpios e integrados, calculamos los indicadores que el gerente necesita para entender la caída en ventas.

-23%
Variación de ventas Feb→Mar
(Mar - Feb) / Feb × 100
S/387
Ticket promedio marzo
SUM(monto) / COUNT(orders)
Lima
Ciudad con mayor caída
GROUP BY ciudad ORDER BY Δ%
68%
Tasa de completitud de datos
Filas limpias / Total filas
SQL · KPIs con Window Functions
-- Variación mes a mes por ciudad (KPI de caída) WITH ventas_mes AS ( SELECT ciudad, mes_año, SUM(monto_soles) AS ventas FROM novamart.analytics.v_ventas_limpias GROUP BY 1, 2 ) SELECT ciudad, mes_año, ventas, LAG(ventas) OVER (PARTITION BY ciudad ORDER BY mes_año) AS ventas_mes_anterior, ROUND( (ventas - LAG(ventas) OVER (PARTITION BY ciudad ORDER BY mes_año)) / LAG(ventas) OVER (PARTITION BY ciudad ORDER BY mes_año) * 100, 2 ) AS variacion_pct FROM ventas_mes ORDER BY variacion_pct ASC;

Semana 02 · Ejercicio Práctico

Tu turno:
resuelve el caso

Usando los datos de NovaMart, responde estas preguntas con Python y SQL en BigQuery. Cada una tiene una pista para guiarte.

Q1
¿Cuántos pedidos duplicados hay en ventas_raw? ¿Qué porcentaje representan del total?
💡 Pista: usa df.duplicated(subset=['order_id']).sum() en Pandas o COUNT(*) - COUNT(DISTINCT order_id) en SQL.
Q2
¿Qué ciudad tiene la mayor caída en ventas de febrero a marzo? ¿Es real o es un problema de datos?
💡 Pista: primero verifica si la caída persiste después de limpiar duplicados y nulos. Puede ser un artefacto de datos sucios.
Q3
Calcula el ticket promedio por categoría de producto en marzo. ¿Qué categoría jalón hacia abajo el promedio general?
💡 Pista: usa GROUP BY categoria con AVG(monto_soles), luego compara contra el promedio total.
Q4
¿Qué pasaría si NO hubieras convertido los montos en dólares a soles? ¿En cuánto estarían sobreestimadas las ventas?
💡 Pista: calcula SUM(monto) sin conversión vs SUM(monto_soles) con conversión. La diferencia es el error que el gerente hubiera visto.
Q5
¿Cuántos clientes compraron en más de una ciudad? ¿Qué implica esto para el análisis geográfico?
💡 Pista: usa COUNT(DISTINCT ciudad) agrupado por cliente_id, luego HAVING COUNT > 1.
