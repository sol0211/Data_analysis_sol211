02
▶ Concepto fundamental

¿Qué es la calidad
de datos?

Un dato tiene calidad cuando es apto para el propósito con el que fue recopilado: correcto, completo, actualizado y consistente.

🎯
Exactitud
¿El dato refleja la realidad?
Mal: Ventas = $1,200,000 (eran $120,000)
📋
Completitud
¿Faltan campos críticos?
Mal: Clientes sin edad ni región
🔄
Consistencia
¿Coincide entre sistemas?
Mal: CRM dice "Activo", ERP dice "Baja"
⏰
Actualidad
¿Son datos vigentes?
Mal: Precios de hace 8 meses
🚫
Sin duplicados
¿El mismo registro existe varias veces?
Mal: Cliente "Ana García" aparece 3 veces
⚖️
Sin sesgo
¿Los datos representan la realidad completa?
Mal: Solo encuestaron clientes satisfechos

04
▶ Caso Real #1 · Retail

El error de Target
en Canadá

Un caso clásico de cómo datos incorrectos destruyen millones en operaciones reales.

🏪 El contexto
Target · Expansión a Canadá · 2013

Target abrió 133 tiendas en Canadá en 2013. Para gestionar el inventario, migró datos de productos desde múltiples sistemas heredados a un nuevo ERP (SAP).

El Problema de Datos

Los datos de inventario tenían errores masivos: dimensiones de productos incorrectas, precios en múltiples monedas sin conversión, campos vacíos, y registros duplicados.

Error 01
Migración defectuosa
Dimensiones de productos erróneas: cajas que "no cabían" en los camiones según el sistema
Error 02
Pedidos inflados
El sistema pedía 99,999 unidades de artículos por error en campos vacíos
Error 03
Estanterías vacías
El sistema decía "hay stock" pero las tiendas estaban físicamente vacías
Consecuencia
Cierre total
Pérdida de $2 mil millones. Target cerró todas sus tiendas en Canadá en 2015

El sistema de seguimiento de COVID usaba archivos Excel para consolidar datos de laboratorios. Excel tiene un límite de columnas (XLS: 65,536 filas). Al llegar al límite, los datos adicionales se descartaban silenciosamente.

06
▶ Anatomía del problema

Cómo un error
escala hasta el desastre

Los problemas de datos rara vez destruyen una empresa solos. Se acumulan en cadena hasta que impactan una decisión crítica.

Origen
El dato entra mal
Error de captura manual, sistema sin validación, migración defectuosa, o fuente no confiable.
Propagación
El dato viaja sin revisión
Se copia a otros sistemas, joins con otras tablas, se incluye en reportes automáticos. Nadie lo cuestiona.
Análisis
El analista confía en el dashboard
El KPI "parece razonable". Nadie verifica la fuente. Se prepara el informe para gerencia.
Decisión
La dirección decide con datos falsos
Lanzamiento de producto, apertura de sucursal, contratación masiva, o reducción de inventario.
Consecuencia
💥 El error se vuelve visible... tarde
Pérdidas financieras, inventarios erróneos, multas regulatorias, daño a la reputación.
07
▶ Buenas prácticas

¿Qué se pudo
haber evitado?

Estas son las prácticas que habrían salvado millones en cada uno de los casos que vimos.

✓
Validación en origen (Data Entry Controls)
Evita que un campo reciba valores negativos de ventas o textos donde van números. Se implementa en formularios, APIs y pipelines ETL.
✓
Perfilado de datos antes de migrar
Analizar la calidad de datos ANTES de moverlos a un nuevo sistema. Target no hizo esto con sus datos de inventario.
✓
Estandarización de definiciones
Acordar qué significa cada campo antes de integrarlo. UK COVID: cada laboratorio tenía su propia definición de "caso confirmado".
✓
Detección automática de duplicados
Herramientas como deduplicación por nombre+email+teléfono evitan que el mismo cliente aparezca 3 veces inflando métricas.
✓
Alertas de anomalías (outlier detection)
Si una venta individual supera el 10x del promedio histórico, el sistema levanta una alerta antes de que llegue al reporte.
✗
Confiar ciegamente en el dashboard
Un dashboard visualmente impresionante no garantiza datos correctos. Siempre preguntar: ¿de dónde vienen estos datos? ¿cuándo fue la última actualización?

▶ Cierre de lección

Lo que aprendimos
hoy

🔍
Calidad de datos
Es la medida en que los datos son exactos, completos, consistentes y actuales para su propósito.
🐛
Tipos de error
Nulos, duplicados, inconsistencias, outliers y sesgos. Cada uno puede distorsionar el análisis.
🏪
Caso Target
Migración defectuosa → inventarios erróneos → $2B perdidos y cierre de 133 tiendas.
🦠
Caso COVID
Excel con límite de filas → 16,000 casos no reportados → decisiones de salud basadas en datos falsos.
🛡️
Prevención
Validar en origen, perfilar antes de migrar, estandarizar definiciones y monitorear anomalías.
🎓 Mensaje para llevar

Como analista de datos, tu responsabilidad no termina en hacer gráficos bonitos. Comienza con entender de dónde vienen los datos. Un modelo brillante sobre datos sucios da respuestas incorrectas con mucha confianza. El trabajo más valioso suele ser el más invisible: limpiar, validar y cuestionar los datos antes de analizarlos.
